# openWa-scripts

Sends a mood-lifting WhatsApp message to a contact twice a day, generated by Gemini AI in casual Spanish. Runs on a schedule via crontab or manually on demand.

## How it works

1. Reads config from `.env`
2. Calls Gemini 2.5 Flash to generate a short, casual Spanish message
3. Sends it via a locally-running [open-wa](https://www.open-wa.org/) instance using an active WhatsApp session
4. Appends the result to `messages.log`

## Prerequisites

- Node.js (version pinned in `.nvmrc` — use `nvm use` to activate)
- A running [open-wa](https://www.open-wa.org/) instance with an **already created and active session**
- A [Gemini API key](https://aistudio.google.com) (free tier)

## Setup

```bash
nvm use
npm install
cp .env.example .env
```

Fill in `.env`:

```
GEMINI_API_KEY=        # from aistudio.google.com
OPENWA_API_KEY=        # from your open-wa dashboard
OPENWA_BASE_URL=       # e.g. http://localhost:2785
OPENWA_SESSION_ID=     # UUID of the active session (GET /api/sessions)
WIFE_CHAT_ID=          # WhatsApp ID format: countrycode+number@c.us
```

To find the session ID:
```bash
curl http://localhost:2785/api/sessions -H "X-API-Key: <your-key>"
```

## Running manually

```bash
npm run send
```

Check the result:
```bash
tail -f messages.log
```

## Automated schedule

Two cron entries fire the script daily via `run.sh`, which sources nvm and reads the Node version from `.nvmrc`:

| Time | Cron expression |
|------|----------------|
| 9:00 AM | `0 9 * * *` |
| 7:00 PM | `0 19 * * *` |

To view the installed cron:
```bash
crontab -l
```

Errors at the cron launch level (e.g. nvm not found) are written to `cron-errors.log`. Sent messages and application errors are written to `messages.log`.

To update the schedule, run `crontab -e`. If you upgrade Node via nvm, update `.nvmrc` with the new version — the cron entries themselves never need to change.
