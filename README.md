# wife

A Spanish-language WhatsApp mood-lifter for your partner, generated by Gemini and sent through an OpenWA gateway on a cron schedule. Driven by a Commander CLI and an Ink TUI; everything runs as docker-compose services.

## Architecture

```
docker compose stack
├── openwa-api        (OpenWA REST API, :2785)        — from ../Deckasoft/OpenWA
├── openwa-dashboard  (OpenWA session manager, :2886) — from ../Deckasoft/OpenWA
├── scheduler         (Ofelia cron, reads data/ofelia.ini)
└── sender            (on-demand CLI/TUI, profile=tools)
```

The `sender` service is **not started by `docker compose up`** — invoke it ad hoc via `bin/wife` (recommended) or `docker compose run --rm sender …` directly. Ofelia fires `wife send` on the schedule defined in `data/ofelia.ini`, spawning a fresh sender container for each run.

Source of truth lives in three files under `data/`:
- `data/config.json` — base URL, session ID, chat ID (Zod-validated, edited via CLI/TUI)
- `data/prompt.json` — system prompt + few-shot examples
- `data/schedule.json` — cron jobs (also rendered to `data/ofelia.ini` on save)

Secrets stay in `.env` (gitignored), exposing only `GEMINI_API_KEY` and `OPENWA_API_KEY`.

## Prerequisites

- Docker Desktop
- The Deckasoft [OpenWA](https://github.com/rmyndharis/OpenWA) repo cloned at `../Deckasoft/OpenWA` (relative to this repo)
- A [Gemini API key](https://aistudio.google.com) (free tier)

## Setup

```bash
cp .env.example .env
# edit .env to fill GEMINI_API_KEY and OPENWA_API_KEY

docker compose up -d
```

This starts `openwa-api`, `openwa-dashboard`, and `scheduler`. Open the dashboard at <http://localhost:2886>, create a session, and scan the QR code from WhatsApp.

Make `wife` available on your PATH:

```bash
ln -s "$(pwd)/bin/wife" ~/.local/bin/wife    # or add ./bin to PATH
```

Seed the config files and fill in the runtime values:

```bash
wife init
wife config set openwaBaseUrl http://openwa-api:2785
wife config set openwaSessionId <session-uuid-from-dashboard>
wife config set wifeChatId <countrycode+number>@c.us
```

## Usage

```bash
wife tui                       # interactive control panel (5 screens)
wife send                      # generate + send right now
wife preview                   # generate + print, no send
wife preview -n 3              # 3 candidates in one shot

wife history --tail 20         # last 20 entries from messages.log
wife history --grep error

wife config list               # show config and (masked) secrets
wife config get wifeChatId
wife config set wifeChatId 521234567890@c.us

wife prompt show               # current system prompt + examples
wife prompt edit               # opens data/prompt.json in $EDITOR
wife prompt add-example "..."
wife prompt remove-example 3

wife schedule list             # current Ofelia jobs
wife schedule add lunch "30 12 * * *"
wife schedule remove lunch
# After any schedule change:
docker compose restart scheduler
```

## TUI

`wife tui` opens a five-tab control panel. `1`-`5` switch tabs, `q` quits.

| Tab | Purpose |
|---|---|
| 1. Home | Status (secrets/config/last send), `p` to preview, `s` to send |
| 2. History | Scrollable view of `messages.log` (↑/↓, j/k, PgUp/PgDn) |
| 3. Settings | Edit `config.json` and `.env` values inline |
| 4. Prompt | Read system prompt, add/remove few-shot examples |
| 5. Schedule | Add/remove Ofelia jobs |

## Local development (no Docker)

```bash
npm install
npm run wife -- --help     # equivalent to running `wife`
npm run send               # tsx src/cli/index.ts send
npm run tui                # tsx src/cli/index.ts tui
npm test                   # vitest
npm run build              # tsup → dist/cli/index.js
```

In local mode, `data/` resolves to `./data/` relative to the repo root. Set `WIFE_DATA_DIR` to override.

## Migrating from the old script

If you're coming from the previous host-cron + `run.sh` setup:

1. Remove the cron entries: `crontab -e` and delete the two `run.sh` lines.
2. Edit `.env` to keep only `GEMINI_API_KEY` and `OPENWA_API_KEY`; drop the rest.
3. Run `wife init` and configure via `wife config set …`.
4. `docker compose up -d` and you're done.
